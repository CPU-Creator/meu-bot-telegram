# Contributing

Obrigado por contribuir com o projeto!

Guidelines rápidas:

- Abra issues descrevendo bugs ou melhorias antes de PRs grandes.
- Crie branches por feature: `feat/descrição` ou `fix/descrição`.
- Execute testes localmente: `PYTHONPATH=. pytest -q`.
- Mantenha o estilo de código consistente; use `black`/`flake8` se disponível.
- Para mudanças que alteram comportamento, adicione/atualize testes.

PRs serão revisados e integrados via CI.
